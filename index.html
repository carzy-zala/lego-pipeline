THE BLUEPRINT

Two raw data sources — LDraw 3D geometry files and Rebrickable catalogue CSVs — are individually unusable for AI model training. This pipeline bridges the gap.

What It Does

An end-to-end data pipeline that ingests LDraw geometry (.dat files) and Rebrickable catalogue data, processes it through a Medallion Architecture (Bronze → Silver → Gold), and produces enriched JSON datasets ready for AI/ML model training.

Why It Was Built

LDraw has geometry but no semantics. Rebrickable has metadata but no geometry. The gap between them makes both sources individually unusable for training 3D AI models. This pipeline solves that fragmentation.

How It Works

Data flows through five stages: Ingestion of raw sources, Bronze layer for schema enforcement, Silver for geometry resolution and metadata joining, Gold for quality filtering, and ML-Ready for feature engineering and export.

Stage 1: Raw Chaos — Bronze Ingestion

Raw .dat geometry files from LDraw and catalogue CSVs from Rebrickable arrive unvalidated. The Ingestion layer performs raw landing — accepting LDraw and Rebrickable sources as-is into the Bronze zone.

Bronze Layer

Stage 2: Schema Detection

Schema enforcement and deduplication begin. The pipeline validates column types, removes duplicate entries, and ensures every record conforms to the expected structure before advancing.

Schema Enforced

Stage 3: Cleaning the Bricks — Silver Layer

Geometry resolution and metadata joining occur here. Invalid rows are flagged with is_valid and issues columns — never dropped. The recursive sub-part resolver ensures geometrically complete point clouds.

Silver Layer

Stage 4: Building Intelligence — Gold Layer

High-quality filtered records emerge. Feature engineering enriches each brick with part category, color name, year of production, and normalized geometry. The Gold layer filters to valid records for model output while preserving full lineage.

Gold Layer

Stage 5: ML Ready — Output

Eight enriched JSON files per brick. NumPy point clouds of shape (N, 3) — N vertices with x, y, z coordinates. Directly consumable by PointNet and similar 3D ML architectures. Point cloud sizes range from 120 to 408 vertices.

MEDALLION ARCHITECTURE

Every great LEGO build starts with a blueprint. Every great data pipeline starts with architecture. Bronze → Silver → Gold.

THE MEDALLION ARCHITECTURE

STEP 01

Bronze Layer

  • Raw landing zone for LDraw .dat files and Rebrickable CSVs
  • Schema enforcement and column type validation
  • Deduplication of incoming records
  • Immutable storage — raw data preserved as-is
  • Timestamp and source tracking for every ingested file
STEP 02

Silver Layer

  • Recursive sub-part geometry resolution
  • Metadata joining via part ID and color code keys
  • Invalid rows flagged with is_valid and issues columns
  • Mesh extraction and point cloud generation
  • Origin centering and unit sphere normalization
STEP 03

Gold Layer

  • High-quality filtered records for model output
  • Feature engineering: category, color, year enrichment
  • Eight enriched JSON files per brick
  • NumPy point clouds — shape (N, 3), 120–408 vertices
  • Directly consumable by PointNet architectures

BUILD LOG

1

Recursive Sub-Part Resolution

LDraw uses a recursive sub-part reference system. A single brick file references stud geometry files, cavity files, and primitives. Our recursive resolver with visited-set circular reference guard and four-location library search ensures every vertex is accounted for while preventing infinite loops.

2

Data Quality: Flagged, Not Dropped

Invalid rows are never dropped — they are flagged with is_valid and issues columns at the Silver layer. The Gold layer filters for valid records for model output, but full lineage is preserved. Every training run is auditable.

3

Precise Geometry Processing

We extract raw triangle meshes from LDraw geometry and perform uniform Poisson disk sampling to generate high-density point clouds. Every asset is centered at the origin and scaled to fit within a unit sphere for consistent ML input.

4

Metadata Fusion & Enrichment

Joining raw geometry with Rebrickable's semantic catalogue requires precise mapping. We use part IDs and color codes as primary keys to fuse 3D data with real-world metadata — part category, color name, and year of production.

5

ML-Ready Output

Eight enriched JSON files per brick. NumPy point clouds of shape (N, 3) — directly consumable by PointNet and similar 3D ML architectures. Point cloud sizes range from 120 to 408 vertices.

6

Future-Proof Architecture

The Medallion architecture accommodates new data modalities — photogrammetry scans, voxel data, video datasets — as new Bronze source modules without restructuring Silver or Gold layers.
